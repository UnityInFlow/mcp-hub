```rust
/// Parse a DaemonResponse from a Status request into a Vec<ServerLiveInfo>.
///
/// Returns Err if the response indicates failure or has unexpected structure.
pub fn parse_live_info(response: &crate::control::DaemonResponse) -> anyhow::Result<Vec<ServerLiveInfo>> {
    anyhow::ensure!(response.ok, "Daemon status query failed: {:?}", response.error);
    let data = response.data.as_ref()
        .ok_or_else(|| anyhow::anyhow!("Empty status response from daemon"))?;
    let servers = data.as_array()
        .ok_or_else(|| anyhow::anyhow!("Expected array from status response"))?;

    servers.iter().map(|s| {
        Ok(ServerLiveInfo {
            name: s["name"].as_str().unwrap_or_default().to_string(),
            state: s["state"].as_str().unwrap_or("unknown").to_string(),
            tool_names: s["tool_names"].as_array()
                .map(|arr| arr.iter().filter_map(|v| v.as_str().map(String::from)).collect())
                .unwrap_or_default(),
            resource_count: s["resources"].as_u64().unwrap_or(0) as usize,
            prompt_count: s["prompts"].as_u64().unwrap_or(0) as usize,
        })
    }).collect()
}
```

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-05 | Spoofing | Unix socket IPC for --live | accept | Socket is in user's config dir with user-only permissions (set by daemon.rs). Local-only attack surface. Same trust model as `mcp-hub status`. |
| T-05-06 | Information Disclosure | tool_names in IPC response | accept | Tool names are not secrets — they are the public API surface of MCP servers. Output goes to local stdout. |
| T-05-07 | Input Validation | DaemonResponse parsing in parse_live_info | mitigate | All JSON field access uses `.as_str().unwrap_or_default()` and `.as_array().unwrap_or_default()` patterns — no panics on malformed response. `response.ok` checked first with `anyhow::ensure!`. |
