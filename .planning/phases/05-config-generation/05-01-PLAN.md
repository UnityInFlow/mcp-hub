`pub fn render_claude_config(config: &HubConfig, live_info: Option<&[ServerLiveInfo]>) -> anyhow::Result<String>`:
- Header: `// Generated by mcp-hub v{CARGO_PKG_VERSION} at {Utc::now().to_rfc3339_opts(chrono::SecondsFormat::Secs, true)}`
- Second line (Claude-specific): `// Paste into mcpServers in ~/.claude.json or .mcp.json`
- Sort server names alphabetically: `let mut names: Vec<&String> = config.servers.keys().collect(); names.sort();`
- For each server:
  - If `transport == "http"`: skip from mcpServers JSON, but add `// WARNING: '{name}' uses http transport -- manual URL config required` before the JSON
  - Call `config::resolve_env(server)?` to get final env map
  - Build `serde_json::json!({"command": server.command, "args": server.args, "env": resolved_env})`
  - If live_info is Some and contains info for this server:
    - If server is not running: inject `// WARNING: {name} is not running (state: {state})` comment
    - If server has tools: inject `// {name}: tools=[{tool_names joined by ", "}]` comment
- Build final JSON: `json!({"mcpServers": mcp_servers_map})`
- Use `serde_json::to_string_pretty(&output_json)?` (default 2-space indent)
- Inject per-server comments by string manipulation: after `"<server_name>": {`, insert comment line(s)
- Return header + JSON + trailing newline

`pub fn render_cursor_config(config: &HubConfig, live_info: Option<&[ServerLiveInfo]>) -> anyhow::Result<String>`:
- Identical logic to render_claude_config except header line 2 reads: `// Paste into ~/.cursor/mcp.json or .cursor/mcp.json`
- Internally call a shared `render_config_internal(config, live_info, header_lines)` to avoid duplication

Private helpers:
- `fn iso_timestamp() -> String` — returns `Utc::now().to_rfc3339_opts(chrono::SecondsFormat::Secs, true)` (e.g. `2026-04-08T14:30:00Z`)
- `fn render_config_internal(config: &HubConfig, live_info: Option<&[ServerLiveInfo]>, format_hint: &str) -> anyhow::Result<String>` — shared rendering logic
- `fn inject_live_comments(json_str: &str, live_info: &[ServerLiveInfo]) -> String` — inserts `//` comment lines into pretty-printed JSON after each server's opening `{` line
- `fn build_warnings(config: &HubConfig) -> Vec<String>` — collects warning lines for http transport servers

**4. src/main.rs:** Add `mod gen_config;` to the module declarations (after `mod control;`, alphabetical is fine). Add match arm in `async_main`:

```rust
Commands::GenConfig(args) => {
    let config = config::find_and_load_config(cli.config.as_deref())
        .context("Failed to load config")?;

    if config.servers.is_empty() {
        eprintln!("\u{26a0} No servers configured in mcp-hub.toml");
    }

    let live_info: Option<Vec<gen_config::ServerLiveInfo>> = if args.live {
        let sock = daemon::socket_path()?;
        let response = control::send_daemon_command(
            &sock,
            &control::DaemonRequest::Status,
            5,
        ).await?;
        Some(gen_config::parse_live_info(&response)?)
    } else {
        None
    };

    let output = match args.format.as_str() {
        "claude" => gen_config::render_claude_config(&config, live_info.as_deref())?,
        "cursor" => gen_config::render_cursor_config(&config, live_info.as_deref())?,
        other => anyhow::bail!(
            "Unknown format '{other}'. Use --format claude or --format cursor."
        ),
    };

    print!("{output}");
    Ok(())
}
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| TOML config file -> gen-config output | Config file is user-controlled local file; output goes to stdout |
| env_file on disk -> output JSON | env_file contents read from disk and included in output |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Information Disclosure | gen_config.rs env output | accept | Per D-08: env var values passed through as-is deliberately. User owns their config file and env_file. Output goes to local stdout. No network exposure. |
| T-05-02 | Tampering | TOML config file | accept | Config is a local file under user control. gen-config only reads, never writes. No privilege escalation vector. |
| T-05-03 | Denial of Service | resolve_env reading env_file | mitigate | resolve_env propagates file-not-found errors with `?` and context message. No infinite loops or unbounded reads (file read is bounded by OS limits). |
| T-05-04 | Input Validation | --format flag | mitigate | Match arm validates format against "claude" and "cursor" only; unknown values produce `anyhow::bail!` with usage hint. No catch-all `_` arm. |
</threat_model>
