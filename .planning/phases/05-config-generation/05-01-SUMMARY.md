# Phase 05 Plan 01: gen-config Subcommand (Offline Mode) Summary

Implemented `mcp-hub gen-config --format claude|cursor` — offline config snippet generator that reads the TOML config and produces ready-to-paste mcpServers JSON blocks for Claude Code and Cursor, with ISO 8601 timestamp header, alphabetical server ordering, HTTP transport exclusion with warnings, and env_file value merging.

## What Was Built

`src/gen_config.rs` — pure rendering module with:
- `render_claude_config(config, live_info)` — Claude Code mcpServers block

- `parse_live_info(response)` — parses daemon Status response into ServerLiveInfo vec (tool_names empty until Plan 02)
- `ServerLiveInfo` struct with name, state, tool_names, resource_count, prompt_count

`tests/gen_config_test.rs` — 8 CLI integration tests using assert_cmd + predicates + tempfile.

## Known Stubs

- `ServerLiveInfo.tool_names` is always `Vec::new()` in `parse_live_info` — the daemon Status response does not yet include tool names. This is intentional and tracked for Plan 02 (IPC protocol extension for --live mode).
- `ServerLiveInfo.resource_count` and `prompt_count` are always `0` — same reason.

These stubs do NOT affect the plan's primary goal (offline config generation without `--live`). The `--live` flag path is wired but will produce empty tool annotation comments until Plan 02 extends the IPC protocol.

## Threat Flags

No new network endpoints, auth paths, or trust boundary surface introduced. The gen-config subcommand reads local files (TOML config, env_file) and writes to stdout. The `--live` path connects to the existing Unix domain socket (already in threat model). No new threat surface beyond what the plan's threat model covers.
