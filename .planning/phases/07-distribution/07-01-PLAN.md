Prepare the codebase for cross-platform compilation and crates.io publishing.

**src/control.rs changes (addresses review concern: Windows Unix sockets):**

The entire module uses `tokio::net::UnixListener` and `tokio::net::UnixStream` which do not exist on Windows. Wrap the module contents with `#[cfg(unix)]` and provide `#[cfg(not(unix))]` stubs.

At the top of `control.rs`, restructure as follows:

- Keep the `DaemonRequest`, `DaemonResponse`, and `DaemonState` type definitions unconditional (they use only serde/tokio types that compile everywhere).
- Wrap `run_control_socket`, `handle_connection`, `dispatch_request`, and `send_daemon_command` with `#[cfg(unix)]`.
- Add `#[cfg(not(unix))]` stub implementations for `run_control_socket` and `send_daemon_command` that return `anyhow::bail!("Unix domain socket IPC is not supported on this platform")`.

Specifically:
- The `use tokio::net::UnixListener;` import at line 12 must move inside the `#[cfg(unix)]` block.
- The `use tokio::io::{AsyncBufReadExt, AsyncWriteExt, BufReader};` import must also move inside the `#[cfg(unix)]` block (only used by socket functions).
- `DaemonRequest`, `DaemonResponse`, `DaemonState` struct definitions stay unconditional — they are referenced by main.rs in non-socket contexts (e.g., constructing status responses).

The `#[cfg(not(unix))]` stubs:
```rust
#[cfg(not(unix))]
pub async fn run_control_socket(_sock_path: &Path, _state: Arc<DaemonState>) -> anyhow::Result<()> {
    anyhow::bail!("Unix domain socket IPC is not supported on this platform. Use foreground mode.")
}

#[cfg(not(unix))]
pub async fn send_daemon_command(
    _sock_path: &Path,
    _request: &DaemonRequest,
    _timeout_secs: u64,
) -> anyhow::Result<DaemonResponse> {
    anyhow::bail!("Unix domain socket IPC is not supported on this platform. Use foreground mode.")
}
```

**src/main.rs changes:**

No structural changes needed. `main.rs` already guards daemon-mode code paths with `#[cfg(unix)]` / `#[cfg(not(unix))]` blocks. The non-daemon subcommands (Stop, Restart, Status, Logs, Reload) call `control::send_daemon_command` unconditionally, but since the `#[cfg(not(unix))]` stub returns an error, these commands will bail with a clear message on Windows. This is correct behavior — those commands require a running daemon which cannot exist on Windows.
