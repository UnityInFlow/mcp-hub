Create `.github/workflows/release.yml` with the following structure. This addresses all Codex review HIGH concerns.

**Trigger (per D-01):**
```yaml
on:
  push:
    tags: ['v*']
```

**Job 1: `preflight` (addresses review concern: tag/version consistency)**
Runs on `[arc-runner-unityinflow]`. Steps:
1. Checkout code.
2. Extract version from tag: `TAG=${GITHUB_REF##refs/tags/}; VERSION=${TAG#v}`
3. Extract version from Cargo.toml: `CARGO_VERSION=$(grep '^version' Cargo.toml | head -1 | sed 's/.*"\(.*\)"/\1/')`
4. Compare: if `$VERSION != $CARGO_VERSION`, fail with clear error message.
5. Run `cargo publish --dry-run` (addresses review concern: dry-run before release).
6. Set output `version` for downstream jobs.

**Job 2: `test` (per D-02)**
Needs: `preflight`. Runs on `[arc-runner-unityinflow]`. Steps:
1. Checkout code.
2. `cargo test`
3. `cargo clippy -- -D warnings`
4. `cargo fmt --check`

**Job 3: `build-linux-x64` (per D-03)**
Needs: `test`. Runs on `[arc-runner-unityinflow]`. Steps:
1. Checkout code.
2. Install cross: `cargo install cross@0.2.5 --locked` (addresses review concern: pin cross version).
3. Build Linux x86_64 (native): `cross build --release --target x86_64-unknown-linux-gnu`
4. **Smoke test Linux binary** (addresses review concern: no binary smoke tests):
   ```bash
   ./target/x86_64-unknown-linux-gnu/release/mcp-hub --version
   ```
5. Build Windows x86_64 (cross): `cross build --release --target x86_64-pc-windows-gnu`
6. **Smoke test Windows binary via Wine** (if available) or skip with note:
   ```bash
   if command -v wine &>/dev/null; then
     wine ./target/x86_64-pc-windows-gnu/release/mcp-hub.exe --version
   else
     echo "Wine not available -- Windows binary smoke test skipped (binary built successfully)"
   fi
   ```
7. Package:
   ```bash
   tar czf mcp-hub-linux-x86_64.tar.gz -C target/x86_64-unknown-linux-gnu/release mcp-hub
   zip mcp-hub-windows-x86_64.zip -j target/x86_64-pc-windows-gnu/release/mcp-hub.exe
   ```
8. Generate SHA256 checksums for both archives.
9. Upload artifacts with name `linux-x64-windows`.

**Job 4: `build-linux-arm64` (per D-03)**
Needs: `test`. Runs on `[orangepi]`. Steps:
1. Checkout code.
2. `rustup target add aarch64-unknown-linux-gnu` (ensure target is available).
3. Build: `cargo build --release --target aarch64-unknown-linux-gnu` (native on ARM64, no cross needed).
4. **Smoke test ARM64 binary:**
   ```bash
   ./target/aarch64-unknown-linux-gnu/release/mcp-hub --version
   ```
5. Package: `tar czf mcp-hub-linux-aarch64.tar.gz -C target/aarch64-unknown-linux-gnu/release mcp-hub`
6. Upload artifact with name `linux-arm64`.

**Job 5: `build-macos` (per D-04, D-05)**
Needs: `test`. Runs on `[macos-arm64]`. **timeout-minutes: 30** (addresses review concern: Mac runner offline causes infinite queue -- timeout prevents this. If Mac is offline, job fails after 30 min and release job does NOT proceed, preventing partial release).
Steps:
1. Checkout code.
2. `rustup target add x86_64-apple-darwin` (arm64 is native).
3. Build arm64: `cargo build --release --target aarch64-apple-darwin`
4. Build x86_64: `cargo build --release --target x86_64-apple-darwin`
5. Create universal binary (per D-05):
   ```bash
   lipo -create \
     target/aarch64-apple-darwin/release/mcp-hub \
     target/x86_64-apple-darwin/release/mcp-hub \
     -output mcp-hub
   ```
   Note: name the lipo output `mcp-hub` (not `mcp-hub-universal`) so the Homebrew formula can use `bin.install "mcp-hub"` directly.
6. **Smoke test universal binary:**
   ```bash
   ./mcp-hub --version
   ```
7. Package: `tar czf mcp-hub-macos-universal.tar.gz mcp-hub`
8. Upload artifact with name `macos-universal`.

**Job 6: `release` (addresses review concern: partial release risk)**
Needs: `[build-linux-x64, build-linux-arm64, build-macos]` -- ALL three must succeed (addresses review concern: release requires all platforms, not just linux-x64).
Runs on `[arc-runner-unityinflow]`.
Permissions: `contents: write`.
Steps:
1. Checkout code.
2. Download ALL artifacts with `actions/download-artifact@v4` and `merge-multiple: true` into `artifacts/` directory.
3. **Validate all 4 expected archives exist** (addresses review concern: validate artifacts before release):
   ```bash
   for f in mcp-hub-linux-x86_64.tar.gz mcp-hub-linux-aarch64.tar.gz mcp-hub-windows-x86_64.zip mcp-hub-macos-universal.tar.gz; do
     test -f "artifacts/$f" || { echo "MISSING: $f"; exit 1; }
   done
   ```
4. Generate `SHA256SUMS.txt` for all archives:
   ```bash
   cd artifacts && sha256sum *.tar.gz *.zip > SHA256SUMS.txt
   ```
5. Create GitHub Release (per D-06):
   ```yaml
   - uses: softprops/action-gh-release@v2
     with:
       files: |
         artifacts/mcp-hub-linux-x86_64.tar.gz
         artifacts/mcp-hub-linux-aarch64.tar.gz
         artifacts/mcp-hub-windows-x86_64.zip
         artifacts/mcp-hub-macos-universal.tar.gz
         artifacts/SHA256SUMS.txt
       generate_release_notes: true
       body: |
         ## Installation

         **Homebrew (macOS):**
         ```bash
         brew install unityinflow/tap/mcp-hub
         ```

         **From source:**
         ```bash
         cargo install mcp-hub
         ```

         **Pre-built binaries:**
         Download the appropriate archive below and extract.

         ## Checksums
         See `SHA256SUMS.txt` for verification.
   ```
6. **Update Homebrew tap** (per D-07, D-08):
   ```bash
   TAG=${GITHUB_REF##refs/tags/}
   VERSION=${TAG#v}
   SHA256=$(sha256sum artifacts/mcp-hub-macos-universal.tar.gz | awk '{print $1}')

   git clone "https://x-access-token:${HOMEBREW_TAP_TOKEN}@github.com/unityinflow/homebrew-tap.git" tap-repo
   mkdir -p tap-repo/Formula

   cat > tap-repo/Formula/mcp-hub.rb << FORMULA
   class McpHub < Formula
     desc "PM2 for MCP servers - manage, monitor, and configure your MCP servers"
     homepage "https://github.com/UnityInFlow/mcp-hub"
     version "${VERSION}"
     license "MIT"

     on_macos do
       url "https://github.com/UnityInFlow/mcp-hub/releases/download/${TAG}/mcp-hub-macos-universal.tar.gz"
       sha256 "${SHA256}"
     end

     def install
       bin.install "mcp-hub"
     end

     test do
       system "#{bin}/mcp-hub", "--version"
     end
   end
   FORMULA

   cd tap-repo
   git config user.name "github-actions[bot]"
   git config user.email "github-actions[bot]@users.noreply.github.com"
   git add Formula/mcp-hub.rb
   git commit -m "chore: bump mcp-hub to ${VERSION}"
   git push
   ```
   Use env: `HOMEBREW_TAP_TOKEN: ${{ secrets.HOMEBREW_TAP_TOKEN }}`

7. **Publish to crates.io** (per D-09, D-10) -- this runs LAST, after binaries and Homebrew are done:
   ```yaml
   - name: Publish to crates.io
     run: cargo publish
     env:
       CARGO_REGISTRY_TOKEN: ${{ secrets.CARGO_REGISTRY_TOKEN }}
   ```
   Note: `--dry-run` was already validated in the preflight job. Use plain `cargo publish` (no `--no-verify`) here since the test job already verified the crate builds and tests pass.

**Important workflow-level settings:**
- `env.CARGO_TERM_COLOR: always` at workflow level for readable CI logs.
- No `continue-on-error` on any build job -- all must succeed for release (addresses review concern: partial release risk).
- The macOS job has `timeout-minutes: 30` to prevent infinite queue when the runner is offline (addresses review concern: Mac runner offline). If the Mac runner is offline, the job times out, the release job's `needs` is unsatisfied, and no partial release happens.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| CI runner to GitHub API | Release creation, artifact upload -- requires contents:write permission |
| CI runner to crates.io | Crate publishing -- requires CARGO_REGISTRY_TOKEN secret |
| CI runner to homebrew-tap repo | Formula update -- requires HOMEBREW_TAP_TOKEN PAT |
| User downloading binary | Must verify SHA256 checksum against SHA256SUMS.txt |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-04 | Tampering | Release artifacts | mitigate | Generate SHA256SUMS.txt in release job; users can verify downloads against checksums |
| T-07-05 | Elevation of Privilege | HOMEBREW_TAP_TOKEN | mitigate | Token stored as GitHub secret, used only in release job, scoped to repo+workflow permissions only |
| T-07-06 | Tampering | Tag/version mismatch | mitigate | Preflight job compares git tag version against Cargo.toml version; fails if mismatched |
| T-07-07 | Denial of Service | Mac runner offline | mitigate | timeout-minutes: 30 on macOS job prevents infinite queue; release job requires all builds so no partial release occurs |
| T-07-08 | Information Disclosure | CARGO_REGISTRY_TOKEN in logs | accept | GitHub Actions automatically masks secrets in logs; standard CI security model |
| T-07-09 | Spoofing | Homebrew formula points to wrong binary | mitigate | Formula URL constructed from GITHUB_REF tag; SHA256 computed from actual artifact; any mismatch fails brew install |
</threat_model>
