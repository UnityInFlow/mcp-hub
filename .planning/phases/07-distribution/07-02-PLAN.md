Create `.github/workflows/release.yml` (create `.github/workflows/` directory first if needed).

The workflow implements all 10 locked decisions (D-01 through D-10). Structure:

**Trigger (per D-01):**
```yaml
on:
  push:
    tags: ['v*']
```

**Job 1: `test` (per D-02)**
- runs-on: [arc-runner-unityinflow]
- Steps: checkout, `cargo test`, `cargo clippy -- -D warnings`, `cargo fmt --check`
- This gates all build jobs.

**Job 2: `build-linux-x64` (per D-03)**
- needs: test
- runs-on: [arc-runner-unityinflow]
- Steps:
  1. Checkout
  2. Install cross: `cargo install cross --locked`
  3. Build Linux x86_64 native: `cross build --release --target x86_64-unknown-linux-gnu`
  4. Build Windows x86_64 cross: `cross build --release --target x86_64-pc-windows-gnu`
  5. Package: `tar czf mcp-hub-linux-x86_64.tar.gz -C target/x86_64-unknown-linux-gnu/release mcp-hub` and `zip mcp-hub-windows-x86_64.zip` (use `cd target/x86_64-pc-windows-gnu/release && zip ../../../mcp-hub-windows-x86_64.zip mcp-hub.exe` since zip may not have -j)
  6. Upload artifacts with `actions/upload-artifact@v4`, name: `linux-x64-windows`, paths: both archives

**Job 3: `build-linux-arm64` (per D-03)**
- needs: test
- runs-on: [orangepi]
- Steps:
  1. Checkout
  2. Ensure target: `rustup target add aarch64-unknown-linux-gnu` (may already exist, no-op if so)
  3. Build native: `cargo build --release --target aarch64-unknown-linux-gnu`
  4. Package: `tar czf mcp-hub-linux-aarch64.tar.gz -C target/aarch64-unknown-linux-gnu/release mcp-hub`
  5. Upload artifact, name: `linux-arm64`

**Job 4: `build-macos` (per D-04, D-05)**
- needs: test
- runs-on: [macos-arm64]
- `continue-on-error: true` (per CONTEXT.md specifics — Mac offline should not block Linux/Windows)
- Steps:
  1. Checkout
  2. Add x86_64 target: `rustup target add x86_64-apple-darwin`
  3. Build arm64: `cargo build --release --target aarch64-apple-darwin`
  4. Build x86_64: `cargo build --release --target x86_64-apple-darwin`
  5. Create universal binary (per D-05): `lipo -create target/aarch64-apple-darwin/release/mcp-hub target/x86_64-apple-darwin/release/mcp-hub -output mcp-hub`
  6. Package: `tar czf mcp-hub-macos-universal.tar.gz mcp-hub`
  7. Upload artifact, name: `macos-universal`

**Job 5: `release` (per D-06, D-07, D-08, D-09, D-10)**
- needs: [build-linux-x64, build-linux-arm64, build-macos]
- `if: always() && needs.build-linux-x64.result == 'success'` (release proceeds even if Mac was offline, as long as Linux succeeded)
- runs-on: [arc-runner-unityinflow]
- permissions: contents: write
- Steps:
  1. Checkout
  2. Download all artifacts: `actions/download-artifact@v4` with `merge-multiple: true`, path: `artifacts/`
  3. List artifacts for debugging: `ls -la artifacts/`
  4. Create GitHub Release (per D-06): `softprops/action-gh-release@v2` with `files: artifacts/*`, `generate_release_notes: true`, `fail_on_unmatched_files: false` (so missing macOS archive does not fail the release)
  5. Update Homebrew tap (per D-07, D-08): bash script block that:
     - Extracts TAG and VERSION from GITHUB_REF
     - Checks if `mcp-hub-macos-universal.tar.gz` exists in artifacts/; if not, skip tap update with a warning
     - Computes SHA256 of the macOS archive: `shasum -a 256 artifacts/mcp-hub-macos-universal.tar.gz | awk '{print $1}'`
     - Clones `https://x-access-token:${HOMEBREW_TAP_TOKEN}@github.com/unityinflow/homebrew-tap.git`
     - Creates `Formula/mcp-hub.rb` with the formula template (class McpHub, desc, homepage, version, license, on_macos block with url pointing to GitHub Release asset and sha256, install block with `bin.install "mcp-hub"`, test block with `system "#{bin}/mcp-hub", "--version"`)
     - Git config, add, commit, push
     - Uses env: `HOMEBREW_TAP_TOKEN: ${{ secrets.HOMEBREW_TAP_TOKEN }}`
  6. Publish to crates.io (per D-09, D-10): `cargo publish --dry-run` first, then `cargo publish` with `CARGO_REGISTRY_TOKEN: ${{ secrets.CARGO_REGISTRY_TOKEN }}`

IMPORTANT naming: The lipo output binary should be named `mcp-hub` (not `mcp-hub-universal`) so the Homebrew formula `install` block is simply `bin.install "mcp-hub"` — no rename needed. The ARCHIVE is named `mcp-hub-macos-universal.tar.gz` for clarity, but the binary INSIDE is `mcp-hub`.

Add a comment block at the top of the workflow explaining what it does and listing the required secrets (CARGO_REGISTRY_TOKEN, HOMEBREW_TAP_TOKEN) and required runner labels (arc-runner-unityinflow, orangepi, macos-arm64).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions -> crates.io | CARGO_REGISTRY_TOKEN authenticates publish; compromise = unauthorized crate versions |
| GitHub Actions -> homebrew-tap repo | HOMEBREW_TAP_TOKEN authenticates push; compromise = malicious formula |
| GitHub Release -> end users | Binary artifacts downloaded by users; compromise = supply chain attack |
| cross Docker containers -> build output | Cross-compilation happens inside Docker; compromised image = tampered binary |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-03 | Tampering | CARGO_REGISTRY_TOKEN secret | mitigate | Token stored as GitHub encrypted secret, never logged. Scoped to publish-only (not admin). Token rotation recommended annually. |
| T-07-04 | Tampering | HOMEBREW_TAP_TOKEN secret | mitigate | PAT scoped to minimum required permissions (repo + workflow only). Stored as GitHub encrypted secret. |
| T-07-05 | Elevation of Privilege | cross Docker images | accept | cross uses official Rust cross-compilation images from ghcr.io/cross-rs. Low risk for a dev tool. Pin cross version (0.2.5) to avoid silent updates. |
| T-07-06 | Information Disclosure | CI logs | mitigate | Secrets are masked in GitHub Actions logs by default. No echo of token values in workflow steps. |
| T-07-07 | Denial of Service | Mac runner offline | mitigate | `continue-on-error: true` on build-macos job + `if: always()` on release job with Linux success check. Release proceeds without macOS binary. |
| T-07-08 | Spoofing | Homebrew formula URL | accept | Formula points to official GitHub Release URL. Users verify via `brew audit`. Low risk for a niche dev tool. |
</threat_model>
