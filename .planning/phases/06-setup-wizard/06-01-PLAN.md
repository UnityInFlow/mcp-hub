Implement the `mcp-hub init` interactive setup wizard that prompts the user for server name, command, args, and transport type, then writes a clean TOML server entry to `./mcp-hub.toml` (creating the file if it does not exist, appending if it does).

Purpose: Eliminates manual TOML editing for adding new servers. Users can run `mcp-hub init` and have a working config entry in seconds.

Output: `src/init.rs` module with full wizard logic, CLI wiring, and integration tests proving both create and append behaviors.

3. Create src/init.rs with these public functions:

**`pub fn run_init_wizard() -> anyhow::Result<()>`** — Main entry point:
  - Call `existing_server_names()` to get names from `./mcp-hub.toml` (empty vec if file missing)
  - Prompt sequence (per D-02):
    a. **Name** (required): `Input::<String>::new().with_prompt("Server name").validate_with(...)` — validation rejects empty strings (per D-08) and names already in the existing list (per D-06; error message: "Name '<name>' already exists. Existing servers: <comma-separated list>"). Uses a loop via `interact()` — dialoguer re-prompts on validation failure automatically.
    b. **Command** (required): `Input::<String>::new().with_prompt("Command").validate_with(...)` — rejects empty (per D-08). No path validation (per D-07).
    c. **Args** (optional): `Input::<String>::new().with_prompt("Arguments (comma-separated, Enter to skip)").default("".to_string()).interact()?` — split on comma, trim each, filter out empty strings. If result is empty vec, no args field in TOML.
    d. **Transport**: `Select::new().with_prompt("Transport").items(&["stdio", "http"]).default(0).interact()?` — index 0 = stdio default (per D-02).
  - Call `format_toml_block(&name, &command, &args, &transport)` to build the TOML string
  - Call `write_server_entry(&toml_block)` to append/create the file (per D-05, D-09)
  - Print success: `println!("Added '{}' to ./mcp-hub.toml", name)` (discretion: print confirmation)

**`pub fn existing_server_names() -> Vec<String>`** — Read `./mcp-hub.toml` if it exists, parse with `toml::from_str::<toml::Value>`, extract keys from `servers` table. Return empty vec on any error (file missing, parse error). Do NOT use `load_config` here because a malformed file should not prevent init from running — the wizard should still work and append a valid entry. Log a warning via `tracing::warn!` if the file exists but fails to parse.

**`pub fn format_toml_block(name: &str, command: &str, args: &[String], transport: &str) -> String`** — Build hand-crafted TOML string (per D-09):
  - Start with `\n[servers.{name}]\n`
  - Add `command = "{command}"\n`
  - If args is non-empty: `args = ["arg1", "arg2"]\n` (each element quoted)
  - If transport is NOT "stdio": `transport = "{transport}"\n` (omit default per D-10)
  - Return the string

**`fn write_server_entry(toml_block: &str) -> anyhow::Result<()>`** — (per D-04, D-05, D-09):
  - Target path: `./mcp-hub.toml` (current directory, always)
  - If file exists: read contents, ensure it ends with a newline, append `toml_block`
  - If file does not exist: write `toml_block` (the leading `\n` in format_toml_block handles separation; for new files, trim the leading newline)
  - Use `std::fs::OpenOptions` with `.create(true).append(true)` for the append case, or `std::fs::write` for the create case

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User stdin -> init wizard | User-provided server name, command, args are written to a config file |
| Config file -> filesystem | Wizard writes/appends to ./mcp-hub.toml in the current directory |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-01 | Tampering | init.rs write_server_entry | mitigate | Use OpenOptions with proper mode bits; write atomically where possible. Validate the generated TOML block parses correctly before writing. |
| T-06-02 | Information Disclosure | init.rs format_toml_block | accept | Server config (command, args) is not secret — it is local developer tooling config. No credentials are prompted (env vars deferred per D-03). |
| T-06-03 | Denial of Service | init.rs existing_server_names | accept | Parsing a large/malformed TOML file could be slow, but this is a local CLI tool running on the user's own machine. No network exposure. |
| T-06-04 | Spoofing | init.rs server name input | mitigate | Validate name is non-empty and not a duplicate of existing servers. Name is used as a TOML key — ensure it does not contain characters that would break TOML syntax (quotes, brackets, newlines). Reject names with TOML-unsafe characters. |
| T-06-05 | Tampering | init.rs command input | accept | No path validation per D-07. Command is written as a TOML string value (properly quoted). Users intentionally provide arbitrary commands. |
</threat_model>
